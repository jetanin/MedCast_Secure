# 🏥 Hospital Data Schema — ข้อมูลที่โรงพยาบาลต้องเก็บ

> Data dictionary สำหรับ **MedCast_Secure**
> กำหนดว่าโรงพยาบาลท้องถิ่นแต่ละแห่งต้องเก็บข้อมูลอะไรบ้าง เพื่อให้ AI ในเครื่อง (edge)
> สร้างฟีเจอร์ → เทรนโมเดล → ผลิต **weight** ส่งเข้าศูนย์กลาง (Federated Learning)

---

## หลักการ

```
(date, drug_id, quantity) ──feature engineering──► เทรนโมเดล ──► weight ──(+DP noise)──► ส่งศูนย์กลาง
```

- **ข้อมูลดิบไม่ออกจากโรงพยาบาล** — ส่งแค่ weight ที่เทรนได้
- เก็บเป็น **ยอดรวมต่อวัน (aggregate)** ไม่ใช่ระดับรายคนไข้ → ลดความเสี่ยง PDPA ตั้งแต่ต้นทาง
- ความถี่ในการเก็บ: **รายวัน** (อย่างน้อย); ระบบรองรับรายชั่วโมงด้วย

---

## 1) ตาราง `drug_usage` — การจ่ายยารายวัน 🟢 *จำเป็น*

ตารางหลักที่ใช้ผลิต weight ได้ทันที (1 แถว = ยา 1 กลุ่ม ต่อ 1 วัน)

| ฟิลด์ | ชนิด | จำเป็น | ตัวอย่าง | คำอธิบาย / ใช้ทำฟีเจอร์อะไร |
|------|------|:----:|---------|------------------------------|
| `date` | DATE | ✅ | `2026-06-04` | วันที่จ่ายยา → time features, cyclical (month/dow), lag, rolling ทั้งหมด |
| `drug_id` | STRING | ✅ | `N02BE` | รหัสกลุ่มยา ATC (ดู [atc_drug_groups.csv](../data/clean/atc_drug_groups.csv)) → one-hot |
| `quantity_dispensed` | FLOAT | ✅ | `32` | จำนวนหน่วยยาที่จ่ายออกในวันนั้น → **target (demand)** + ต้นทาง lag/rolling/relational |
| `hospital_id` | STRING | ✅ | `HOSP_017` | รหัสโรงพยาบาล (ใช้แยก client ใน FedAvg + แสดงบนแผนที่) |

**ตัวอย่างไฟล์ (`drug_usage.csv`):**
```csv
date,drug_id,quantity_dispensed,hospital_id
2026-06-04,N02BE,32,HOSP_017
2026-06-04,R03,9,HOSP_017
2026-06-04,N05C,0,HOSP_017
```

> ⚠️ ต้องมีข้อมูลย้อนหลัง **อย่างน้อย ~30 วัน** ก่อนเริ่มเทรน เพราะฟีเจอร์ `lag_28` และ `roll_*_30` ต้องใช้ประวัติ 30 วัน

## 2) ตาราง `inventory` — สถานะคลังยา 🟡 *แนะนำ*

ทำให้ระบบรู้ "สต็อกที่เหลือจริง" → เตือนขาดแคลนได้แม่นขึ้น (ไม่ใช่แค่พยากรณ์ความต้องการ)

| ฟิลด์ | ชนิด | จำเป็น | ตัวอย่าง | ปลดล็อกอะไร |
|------|------|:----:|---------|--------------|
| `date` | DATE | 🟡 | `2026-06-04` | วันที่บันทึกสต็อก |
| `drug_id` | STRING | 🟡 | `N02BE` | รหัสกลุ่มยา ATC |
| `stock_on_hand` | FLOAT | 🟡 | `120` | คงเหลือในคลัง → **days-of-supply = สต็อก ÷ พยากรณ์** = สีเขียว/เหลือง/แดง |
| `reorder_level` | FLOAT | ⬜ | `40` | จุดสั่งซื้อ → trigger สั่งยาอัตโนมัติ |
| `expiry_date` | DATE | ⬜ | `2027-01-15` | วันหมดอายุ → ระบบ **FEFO** หมุนเวียนยา ลดของเสีย |
| `unit` | STRING | ⬜ | `tablet` | หน่วยนับ |

**ตัวอย่างไฟล์ (`inventory.csv`):**
```csv
date,drug_id,stock_on_hand,reorder_level,expiry_date,unit
2026-06-04,N02BE,120,40,2027-01-15,tablet
2026-06-04,R03,18,25,2026-09-30,inhaler
```

## 3) ตาราง `hospital_master` — ข้อมูลโรงพยาบาล 🟡 *แนะนำ (เก็บครั้งเดียว)*

สำหรับ **Overview Map** และตั้ง threshold การขนส่ง

| ฟิลด์ | ชนิด | จำเป็น | ตัวอย่าง | คำอธิบาย |
|------|------|:----:|---------|----------|
| `hospital_id` | STRING | 🟡 | `HOSP_017` | รหัสโรงพยาบาล (เชื่อมกับตารางอื่น) |
| `name` | STRING | ⬜ | `รพ.สต. บ้านหนองคู` | ชื่อ |
| `latitude` | FLOAT | 🟡 | `14.9802` | พิกัด → หมุดบนแผนที่ |
| `longitude` | FLOAT | 🟡 | `102.0978` | พิกัด → หมุดบนแผนที่ |
| `region` | STRING | ⬜ | `ภาคอีสาน` | จัดกลุ่ม/กรองในแดชบอร์ด |
| `lead_time_days` | INT | ⬜ | `2` | เวลาส่งยาจากศูนย์กลาง → ตั้ง threshold เตือนล่วงหน้า |

## สรุปลำดับความสำคัญ

| ระดับ | ตาราง / ฟิลด์ | ได้อะไร |
|------|----------------|---------|
| 🟢 **ขั้นต่ำ** | `drug_usage` (date, drug_id, quantity, hospital_id) | เทรนโมเดล + ผลิต weight ได้ทันที (พยากรณ์ความต้องการ) |
| 🟡 **แนะนำ** | + `inventory.stock_on_hand`, `hospital_master.lat/lon` | เตือน "ขาดแคลนจริง" + แสดงแผนที่ |
| ⬜ **เสริม** | + reorder, expiry, lead_time | สั่งยาอัตโนมัติ, FEFO, เตือนตามเวลาขนส่ง |

---

## หมายเหตุด้านความปลอดภัย (PDPA / Privacy)

1. **ไม่เก็บข้อมูลระบุตัวคนไข้** (ชื่อ, HN, เลขบัตร) — เก็บแค่ยอดรวมต่อวันต่อกลุ่มยา
2. ข้อมูลดิบทั้ง 3 ตาราง **อยู่ในเครื่องโรงพยาบาลเท่านั้น** — ออกไปแค่ weight
3. weight ที่ส่งออกผ่าน **Differential Privacy noise + Secure Aggregation + TLS/SSL**
4. `drug_id` ใช้ระดับ **กลุ่ม ATC** (เช่น N02BE) ไม่ใช่ชื่อการค้า → ลดการระบุพฤติกรรมเฉพาะเจาะจง
